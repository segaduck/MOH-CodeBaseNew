description: 將所有變更提交至本地 Git 並推送到 GitHub，並以繁體中文撰寫詳細說明
---

請幫我完成以下 Git 操作：

1. **檢查目前狀態**：執行 `git status` 查看所有未追蹤和已修改的檔案

2. **檢視變更內容**：執行 `git diff` 查看所有變更的詳細內容

3. **查看提交歷史**：執行 `git log --oneline -5` 了解最近的提交風格

4. **暫存所有變更**：將所有相關變更加入暫存區（排除不應提交的檔案如 bin/、obj/、*.user 等）

5. **撰寫提交訊息**：請以**繁體中文**撰寫詳細的 commit message，格式如下：
   ```
   [類型] 主要變更摘要
   
   詳細說明：
   - 變更項目 1
   - 變更項目 2
   - ...
   
   影響範圍：
   - 受影響的模組/功能
   ```
   
   類型包含：
   - [新增] 新功能
   - [修復] 錯誤修正
   - [優化] 效能或程式碼優化
   - [重構] 程式碼重構
   - [文件] 文件更新
   - [測試] 測試相關

6. **提交變更**：執行 `git commit` 提交所有變更

7. **推送至遠端**：執行 `git push` 將變更推送到 GitHub

請確保：
- 提交訊息清楚描述「為什麼」要做這些變更，而不只是「做了什麼」
- 不要提交敏感資訊（密碼、API 金鑰等）
- 如果是第一次推送分支，使用 `git push -u origin <branch-name>`
